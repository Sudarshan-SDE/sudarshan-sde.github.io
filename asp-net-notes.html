.NET

.NET is a free, open-source, cross-platform, modular, high performance and cloud-optimized developement platform designed and developed by Microsoft to deliver productivity, performance and security for building modern, interactive, dynamic, cloud-enabled applications, Web APIs, microservices. It supports multiple languages, libraries, and frameworks that can run on Windows, Linux, and macOS.

  • C# is the primary programming language for .NET
  • It is not a continuation of ASP.NET 4.
  • 2016 → .NET Core 1.0
  • 2018 → .NET Core 2.0
  • 2019 → .NET Core 3.1
  • 2020 → .NET 5
  • 2021 → .NET 6
  • 2022 → .NET 7
  • 2023 → .NET 8

Key Features of .NET

  • Open Source: .NET Core is a free, open-source platform, meaning you can access the source code and contribute to its development.
  • Cross-Platform: .NET Core was designed from scratch to be cross-platform, running on Windows, Linux, and macOS.
  • High Performance: .NET Core was optimized for high performance, making it suitable for modern, scalable application development.
  • Long Term Support (LTS): .NET Core introduced LTS releases, providing a stable and supported platform for an extended period.
  • MVC Architecture: .NET Core supports the Model-View-Controller design pattern for structured web application development.
  • Razor Views: Uses Razor syntax for building dynamic, server-rendered views.
  • Tag Helpers: Provides HTML tag helpers for generating clean, dynamic HTML content.
  • Robust Security: .NET Core provides built-in features for authentication, authorization, and data protection.
  • Cloud Optimized: .NET Core is designed to be cloud-ready, with first-class support for containerization and microservices.
  • Garbage Collection: Provides automatic memory management via the CLR garbage collector.
  • Dependency Injection: .NET Core has a built-in DI container, promoting loosely coupled and testable code.
  • Middleware Pipeline: .NET Core uses a middleware pipeline to handle HTTP requests and responses in a modular way.

ASP.NET Core

ASP.NET Core is a web development framework provided by .NET used for building web applications, REST APIs, and micro-services.

Difference between .NET Framework and ASP.NET Core

.NET Framework
  • .NET Framework is traditional and older framework.
  • .Net Framework follows MVC design pattern.
  • .Net Framework supports only Windows.
  • .Net Framework introduced in 2002 as .NET Framework 1.0
  • .Net Framework latest version is (4.8.1)
  • .Net Framework is no longer supported.
  • .Net Framework used for Win Forms, WPF, ASP.NET Web Forms, older enterprise apps.
  • .Net Framework does not supports mobile development.
  • .Net Framework is less scalable.
.NET Core
  • .Net Core modern successor .Net framework.
  • .Net Core also follows MVC design pattern.
  • .Net Core is cross-platform.
  • .Net Core introduced in 2016 as .NET Core 1.0
  • .Net Core uses latest version (.NET 6, .NET 8, .NET 9)
  • .Net Core is long-term support.
  • .Net Core used for Web APIs.MVC Apps, Micro services, Cloud apps etc.
  • .Net Core supports mobile development.
  • .Net Core is high scalable.

Model-View-Controller (MVC)

MVC stands for Model-View-Controller. It is an architectural design pattern. This pattern helps to achieve separation of concerns. The entire application is divided into three main components -> Models, Views, and Controllers.
- The model contains the business logic entities and handles data.
- The View contains the UI Presentation logic.
- The Controller is a mediator, it handles HTTP requests and returns responses.

  • Separation of concern
  • Easy to maintain and test
  • Clean code structure

Middleware

.NET introduced a new concept called Middleware. Middlewares are the software components that provides services. In ASP.NET Core, middleware components are used to handle HTTP requests and responses in the request processing pipeline. Middleware sits between the incoming request and the outgoing response.

  • app.UseExceptionHandler("/Error/500");
  • app.UseHttpsRedirection();
  • app.UseStaticFiles();
  • app.UseRouting();
  • app.UseAuthentication();
  • app.UseAuthorization();
  • app.UseEndpoints(endpoints => { endpoints.MapControllers(); });

Dependency Injection

Dependency Injection (DI) is a design pattern that allows us to remove hard-coded dependencies and make our applications more flexible and maintainable. Dependency Injection is a design pattern by injecting dependencies to a class from outside instead of the class creating them itself, helping to achieve loose coupling and better testability.

  • Loose Coupling
  • Easy Unit Testing
  • Better Maintainability
  • builder.Services.AddScoped();

Service Lifetimes

Service Lifetime defines how long a service instance lives and how it is reused by the Dependency Injection container.

  • Transient -> A new instance of the service is created every time it is requested from the dependency injection container.
    builder.Services.AddTransient();
  • Scoped -> A single instance of the service is created per HTTP request and shared within that request.
    builder.Services.AddScoped();
  • Singleton -> Only one instance of the service is created for the entire application lifetime and reused everywhere.
    builder.Services.AddSingleton();

Kestrel Web Server

Kestrel is a cross-platform web server for ASP.NET Core applications. It is the default web server for ASP.NET Core applications and is optimized for high-performance scenarios.

App Settings (appsettings.json)

appsettings.json stores application configuration settings such as connection strings, JWT, logging settings, and custom configuration.

Routing

Routing is the mechanism that maps HTTP requests to the appropriate endpoints, such as controller actions, Razor Pages, or other request handlers.
There are two types of routing:
Conventional Routing: Conventional Routing refers to a routing mechanism where URL patterns are centrally defined. This routing follows a standard convention, typically defined in the program.cs file.
Attribute Routing: Attribute Routing refers to a routing mechanism where URL patterns are defined as attributes on the action methods. This routing provides more flexibility and control over the URL structure.

Controller

A Controller is a class that handles incoming HTTP requests and returns response. It acts as a mediator between business logic and presentation layer.
Controller contains Action methods that process requests.
Controllers inherit from the Controller Base Class provided by the Microsoft.AspNetCore.Mvc namespaces.

Action Method

An Action Methods are the public methods within a Controller that handles a specific HTTP request and returns an appropriate response.
Action Methods can return various types of responses, such as views, JSON data, or status codes, depending on the requirements of the application.
Note: Action Methods must be public and cannot be static.

IActionResult and ActionResult

IActionResult is an interface that allows returning different types of results such as ViewResult, JsonResult, ContentResult, RedirectResult, etc.

public IActionResult Index()
{
    return View();
}

public IActionResult GetData()
{
    return Json(new { name = "Sudarshan" });
}

public IActionResult RedirectPage()
{
    return RedirectToAction("Index");
}

ActionResult is a class that implements IActionResult and provides a convenient way to return different types of responses from an action method.

public ActionResult About()
{
    return View();
}

View

Model Binding is the process of mapping HTTP request data to action method parameters or properties of a model object.
